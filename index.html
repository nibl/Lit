Lit

One billion people worldwide cannot read. We can change that.

Lit is an app to teach reading to adults on smartphones and in web browsers. Technology now gives people the means to learn anywhere, anytime. Learning power is in their pockets. Join the forum and get involved.

We need your help

Here are the main tasks. Keep in mind that users cannot read properly. They do not use the Internet or text-based media the way you do. Everything, including support will need to be audio-visual:

  • Create lessons Details »

    • Lit takes a sound-based (phonetic) approach to reading. Students learn what sounds a letter may stand for, then sounds out words. You do not need any knowledge of linguistics to create lessons. It's knowledge you have gained inherintly while learning to read yourself.
    You may find creating lessons an interesting challenge. Especially in English, where spelling and speech have drifted far apart. It's a lot trickier than you may think. Contribute via our Lesson Pad. It's part of the project Wiki on GitHub. Lesson Pad

  • Code javascript »

    • We are using Javascript so the same code can be used in web browsers and on mobile devices by wrapping the code with PhoneGap (Cordova). Browsers and Android will be the initial platforms, since these have the widest usage and lowest access cost. iOS developers are encouraged to make the necessary changes for iPhone and iPad. Using PhoneGap this should be fairly swift.

  • Design an entirely audio-visual UI/UX »

    • Here's a challenge for a UI designer. Most user interfaces depend on text. Lit cannot. An exception are games interfaces and educational programs for very young children. However, we have to guide adults through an entirely audio-visual environment.

  • Design graphics »

    • The UI graphics need to "talk" to the user even more than usual. We have no text hints to rely on.

  • Record voice-overs »

    • All student instruction is spoken. We need people to record voice-overs. These will be synchronized with the visual elements to provide instructions in lessons, quizes and hints. The texts to be spoken will be in the Lesson Pad.

  • Reach out to potential users »

    • This is possibly the biggest challenge! Think about it. How do we tell people who cannot read text-based media about the program? If we cannot reach the users the project fails. This task is vital to success.
    We can inform people who can read to tell those who cannot. Radio stations are another way. We could ask local radio stations to help get the word out. This will need to be very creative so that radio stations benefit, and it doesn't feel in any way condescending towards functionally illiterate listeners.
